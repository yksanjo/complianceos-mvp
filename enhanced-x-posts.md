### Post 2: Audio Analysis Tech

**Original Post:**
```
The secret sauce behind audio2strudel? üéº

‚ú® Autocorrelation-based pitch detection

**Follow-Up Post 2.1: Pitch Detection Deep Dive**
```
How does audio2strudel detect pitch? üéµ

Autocorrelation algorithm:
1. Compare audio signal with time-shifted version
2. Find peaks in correlation function
3. Calculate fundamental frequency
4. Map to musical notes

Why autocorrelation?
‚Ä¢ Works with monophonic audio
‚Ä¢ Handles noisy recordings
‚Ä¢ Real-time capable
‚Ä¢ No machine learning needed

[Screenshot: Waveform with detected pitches overlaid]

All in JavaScript. All in the browser.

#AudioProcessing #Algorithms
```

**Follow-Up Post 2.2: Key Finding Algorithm**
```
Finding the musical key is harder than you think üéπ

Krumhansl-Schmuckler algorithm:
‚Ä¢ Analyzes pitch class distribution
‚Ä¢ Compares to 24 major/minor key profiles
‚Ä¢ Scores each key based on fit
‚Ä¢ Returns most likely key

Why it matters:
‚Ä¢ Correct chord detection
‚Ä¢ Proper transposition
‚Ä¢ Better pattern generation

[Screenshot: Visualization showing key detection process]

The math is beautiful. The results are accurate.

#MusicTheory #Algorithms
```

**Follow-Up Post 2.3: Real-Time FFT Analysis**
```
Processing audio in real-time with FFT üìä

4096-sample frames:
‚Ä¢ ~93ms at 44.1kHz
‚Ä¢ Fast enough for real-time
‚Ä¢ Detailed enough for accuracy

What we extract:
‚Ä¢ Frequency spectrum
‚Ä¢ Harmonic content
‚Ä¢ Timbre characteristics
‚Ä¢ Onset detection

[Screenshot: FFT visualization showing frequency spectrum]

All with Web Audio API. No external libraries.

#WebAudioAPI #SignalProcessing
```
