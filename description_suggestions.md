## üí° Tips for Great Descriptions

1. **Be specific** - What does it actually do?
2. **Mention the tech** - Include the main language/framework
3. **State the value** - Why would someone use this?
4. **Keep it concise** - Under 350 characters for GitHub
5. **No marketing fluff** - "Revolutionary" and "cutting-edge" are overused

### Template Formulas

**For Tools**:
```
[What it does] for [who]. Built with [tech].
```

**For Apps**:
```
[App type] that [main feature]. [Secondary benefit]. [Tech stack].
```

**For Libraries**:
```
[Language] library for [purpose]. [Key feature].
```
